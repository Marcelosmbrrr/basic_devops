Docker Compose é o orquestrador de containers da Docker.
Imagine o trabalho que seria executar algumas dezenas de containers manualmente na linha de comando.
Docker compose é uma ferramenta para definição e execução de múltiplos containers Docker. Com ela é possível configurar todos os parâmetros necessários para executar cada container a partir de um arquivo de definição.
Dentro desse arquivo, definimos cada container como um serviço.

==== O ARQUIVO DOCKER COMPOSE ====

O arquivo de definição do Docker Compose é escrito com a linguagem YAML.
Esse arquivo serve para definir os eventos e as definições antes e depois da execução dos serviços (vulgo container).
Por exemplo, se a aplicação terá MySQL, PHP e Laravel, cada uma dessas partes será um serviço/container, e é no composer file que todos os parâmetros de funcionamento serão setados.
Esse arquivo por padrão tem como nome docker-compose.yml e deve ser criado no diretório root do projeto.
O padrão YAML utiliza a indentação como separador dos blocos de códigos, assim como na linguagem Python.
Cada linha desse arquivo pode ser definida com uma chave valor ou uma lista.
